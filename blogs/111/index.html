Deeplinking in Flash

Dec 6th, 2006 | Comments

Deeplinking？深度链接？！
反正学电脑应该知道啥意思…普通的HTML网页都有这个东西，最Common的Feature。
可惜在Flash中就不是这么一回事儿了。仿佛Deeplinking在Ajax里边也有缺陷…不过我没研究过Ajax，也不知道怎么一回事儿了。
 
今天看了一下关于Flash Deeplinking的资料，最后实践了一下，基本上成功得在Flash里边添加了Deeplinking，还包括Browser history。下一个网站就只需要把结构分清楚，Deeplinking就不难实现了。
 
http://2Advance.com 给了一个很典型的Deeplinking Flash Website。从它的HTML源码看了一下…
 
<title>2Advanced v5 - Attractor</title>
<script type="text/javascript" src="includes/dhtmlHistory.js"></script>
<script type="text/javascript" src="includes/v5util.js"></script>
<script type="text/javascript" src="includes/swfobject_source.js"></script>
<script src="http://www.google-analytics.com/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
 
用了些乱七八糟的东西…
dhtmlHistory.js应该是一个JavaScript，大概是用来弄Browser Back & forward button的东西…
v5util.js….应该是一个它们工作组自己写的JavaScript…
swfobject_source.js可能就是实现Deeplinking的最关键的JavaScript…
另外剩下一个可能是给google search engine用的一个JavaScript
 
 
不过实际上不用这么麻烦…网上有一个JavaScript Class，外带一个ActionScript Class可以处理这方面的问题….SWFAddress。
SWFAddress is based on SWFObject.
 
SWFObject is a small Javascript file used for embedding Macromedia Flash content. The script can detect the Flash plug-in in all major web browsers (on Mac and PC) and is designed to make embedding Flash movies as easy as possible. It is also very search engine friendly, degrades gracefully, can be used in valid HTML and XHTML 1.0 documents*, and is forward compatible, so it should work for years to come.
 
SWFAddress 借助 SWFObject实现了Flash的Deeplinking和浏览器的Back & Forward button。
具体怎么用，去看官方的介绍
http://www.asual.com/swfaddress/
用起来很方便…还不错….改一下SWF的Embed方式，用SWFObject嵌入Flash…再在Flash ActionScript里边include一个Class就可以用了…
